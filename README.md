# SYK9 减脂打卡助手

这是一个适合新手练习 AI 辅助编程的纯前端网页项目。

项目只使用：

- HTML
- CSS
- JavaScript
- localStorage

没有 React、Vue、Next.js，也没有后端和数据库。

## 文件说明

### `index.html`

网页结构文件，负责放置页面内容。

里面包含：

- 项目标题
- 今日打卡表单
- 今日热量缺口展示
- 累计进度展示
- 历史记录列表

### `style.css`

网页样式文件，负责页面外观。

里面包含：

- 蓝白色主题
- 手机 App 风格布局
- 圆角卡片
- 柔和阴影
- 输入框、按钮、历史记录样式

### `script.js`

网页逻辑文件，负责功能。

里面包含：

- 自动计算今日热量缺口
- 保存今日记录
- 从 localStorage 读取历史记录
- 删除某一天记录
- 计算累计热量缺口
- 按 7700 kcal ≈ 1 kg 脂肪估算累计减重

### `README.md`

项目说明文件，也就是你现在看到的这个文件。

## 怎么打开

最简单的方法：

1. 打开项目文件夹。
2. 双击 `index.html`。
3. 浏览器会直接打开网页。

这是纯前端项目，所以不需要安装任何东西，也不需要启动服务器。

## 怎么测试每个功能

### 1. 测试输入今日体重

在“今日体重 kg”里输入：

```text
79.0
```

页面应该可以正常接收小数。

### 2. 测试输入今日摄入热量

在“今日摄入热量 kcal”里输入：

```text
1800
```

今日热量缺口会自动变化。

### 3. 测试输入今日运动消耗

在“今日运动消耗 kcal”里输入：

```text
230
```

今日热量缺口会再次自动变化。

### 4. 测试保存今日记录

点击“保存今日记录”。

历史记录列表里应该出现今天这条记录。

### 5. 测试刷新后记录不丢失

按浏览器刷新。

如果历史记录还在，说明 localStorage 保存成功。

### 6. 测试删除某一天记录

在历史记录里点击“删除”。

那一天的记录应该从列表里消失，累计热量缺口也会重新计算。

### 7. 测试累计热量缺口

多保存几天不同日期的记录。

“累计热量缺口”会把所有历史记录的缺口加起来。

### 8. 测试估算累计减重

页面会用这个公式估算：

```text
估算累计减重 = 累计热量缺口 ÷ 7700
```

例如累计缺口是 `7700 kcal`，估算累计减重大约就是 `1.00 kg`。

## 当前计算方式

第一版为了适合新手，只做一个简单估算：

```text
基础代谢 = 体重kg × 9.99 + 身高cm × 6.25 - 年龄 × 4.92 + 5
估算维持热量 = 基础代谢 ÷ 0.7 + 今日运动消耗
今日热量缺口 = 估算维持热量 - 今日摄入热量
```

注意：这是练习项目里的粗略估算，不是医学建议。真实减脂更应该看 1-2 周趋势，不要被 1-3 天体重波动影响心态。

## 下一步最适合增加什么功能

最适合的新手下一步是：

1. 增加“运动类型”选择，比如跑步、游泳、引体向上。
2. 给每条记录加一个“备注”，记录今天吃了什么或做了什么运动。
3. 增加 73 kg 目标进度条。
4. 增加体重趋势折线图。

建议下一步先做“运动类型 + 备注”，因为它不难，但会让历史记录更有用。
